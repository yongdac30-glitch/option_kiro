# 期权风险监控系统 - 需求文档

## 1. 项目概述

开发一个基于Web的期权组合风险监控系统，用于实时监控多个期权合约（主要是sell put）在不同市场价格和隐含波动率情况下的潜在盈亏情况。

## 2. 用户故事

### 2.1 持仓管理
**作为** 期权交易者  
**我想要** 通过UI界面添加、查看和删除我的期权持仓  
**以便** 管理我的期权组合

**验收标准：**
- 2.1.1 用户可以通过表单添加新的期权合约，包含以下信息：
  - 标的资产代码/名称
  - 期权类型（Put/Call）
  - 行权价
  - 到期日
  - 合约数量（正数表示买入，负数表示卖出）
  - 开仓价格（权利金）
- 2.1.2 用户可以查看所有当前持仓的列表
- 2.1.3 用户可以删除已平仓或不再需要监控的持仓
- 2.1.4 用户可以编辑现有持仓的信息
- 2.1.5 所有持仓数据持久化存储在数据库中

### 2.2 市场价格输入
**作为** 期权交易者  
**我想要** 手动输入标的资产的当前市场价格  
**以便** 基于最新价格计算盈亏

**验收标准：**
- 2.2.1 UI界面提供输入框用于输入标的资产当前价格
- 2.2.2 价格输入后立即触发盈亏计算更新
- 2.2.3 系统记住每个标的资产的最后输入价格
- 2.2.4 UI预留未来接入实时数据API的扩展接口

### 2.3 隐含波动率自定义
**作为** 期权交易者  
**我想要** 自定义隐含波动率参数  
**以便** 在不同波动率情景下评估风险

**验收标准：**
- 2.3.1 用户可以为每个持仓或整个组合设置隐含波动率
- 2.3.2 系统提供默认的隐含波动率值
- 2.3.3 用户可以创建多个波动率情景进行对比分析
- 2.3.4 波动率变化后立即更新盈亏计算

### 2.4 盈亏分析与可视化
**作为** 期权交易者  
**我想要** 查看在不同价格档位下的组合盈亏情况  
**以便** 了解我的风险敞口

**验收标准：**
- 2.4.1 系统计算并展示当前价格上下50%波动范围内的盈亏
- 2.4.2 价格档位足够细密，能够准确反映盈亏曲线
- 2.4.3 使用图表（盈亏曲线图）直观展示不同价格下的盈亏
- 2.4.4 使用表格展示关键价格点的具体盈亏数值
- 2.4.5 图表和表格实时更新，响应价格和波动率的变化

### 2.5 最大亏损监控
**作为** 期权交易者  
**我想要** 知道理论最大亏损及其对应的价格点  
**以便** 评估最坏情况下的风险

**验收标准：**
- 2.5.1 系统计算并显示理论最大亏损金额
- 2.5.2 系统标识出触达最大亏损的标的价格
- 2.5.3 在盈亏曲线图上突出显示最大亏损点
- 2.5.4 当最大亏损超过用户设定的阈值时提供警告提示

### 2.6 多到期日支持
**作为** 期权交易者  
**我想要** 系统能够处理不同到期日的期权合约  
**以便** 监控整个组合的时间价值衰减

**验收标准：**
- 2.6.1 系统正确计算每个合约距离到期的剩余时间
- 2.6.2 盈亏计算考虑时间价值衰减（Theta效应）
- 2.6.3 用户可以选择查看特定日期的组合盈亏情况
- 2.6.4 系统自动识别并提醒即将到期的合约

## 3. 技术需求

### 3.1 前端技术栈
- React.js
- 图表库（推荐：Recharts 或 Chart.js）
- UI组件库（推荐：Ant Design 或 Material-UI）

### 3.2 后端技术栈
- Python
- Web框架（推荐：FastAPI 或 Flask）
- 期权定价：Black-Scholes模型

### 3.3 数据库
- 关系型数据库（推荐：SQLite用于开发，PostgreSQL用于生产）
- 存储持仓数据、价格历史、用户配置

### 3.4 期权定价模型
- 使用Black-Scholes模型计算期权理论价值
- 支持自定义隐含波动率输入
- 考虑时间价值衰减

## 4. 非功能性需求

### 4.1 性能
- 盈亏计算和图表更新响应时间 < 1秒
- 支持至少50个并发持仓的计算

### 4.2 可用性
- 界面简洁直观，适合快速查看风险状况
- 支持桌面浏览器（Chrome、Firefox、Safari、Edge）

### 4.3 可扩展性
- 预留实时数据API接入接口
- 预留未来添加Greeks指标的扩展点
- 预留多用户支持的架构设计

### 4.4 数据安全
- 持仓数据本地存储
- 预留用户认证机制的扩展点

## 5. 约束条件

- 初始版本仅支持手动输入价格
- 初始版本不包含Greeks指标监控
- 初始版本为单用户系统
- 使用Black-Scholes模型（适用于欧式期权）

## 6. 未来扩展方向

- 接入实时市场数据API
- 添加Greeks指标监控
- 支持多用户和权限管理
- 添加历史盈亏回测功能
- 支持更多期权策略分析（如价差、跨式等）
- 移动端适配
